Moveit Task Constructor

  • 运动跟踪与控制（MTC）的核心思想在于，复杂的运动规划问题可分解为一系列更简单的子问题。最高层级的规划问题被定义为任务（Task），而所有子问题则通过**阶段（Stage）**来定义。阶段的排列顺序和层级结构可任意组合，仅受限于各阶段的类型。阶段排列顺序由结果传递的方向所决定。基于结果流存在三类可能的阶段：生成阶段、传播阶段和连接阶段：
    • 生成器独立于邻近阶段计算结果，并将其向前向后双向传递。例如用于几何姿态的反向运动学采样器，其接近和远离运动（邻近阶段）均依赖于解算结果。
    • 传播器接收相邻阶段的结果，解决子问题后将结果传播至对侧邻居。根据具体实现，传播阶段可分别向前、向后或双向传递解法。例如某个阶段可基于起始状态或目标状态计算笛卡尔路径。
    • 连接器不会传播任何结果，而是试图弥合两个邻接节点状态之间的差距。例如，从给定状态计算到另一个状态的自由运动计划。

主要组件

  • Task：整个流程的顶层容器，由多个stage组成
  • Stage（阶段）：任务中的一个步骤
  • Container（容器）：Stage可以被组合到不同的容器结构中

Stage

Generator Stages（生成阶段）

  • 生成新的机器人状态或候选解
  • 特点
    • 没有输入
    • 直接生成新的规划状态
    • 通常是任务的起点
  • 常见Generator
Stage 作用
CurrentState 获取机器人当前状态
GenerateGraspPose 生成抓取姿态
GeneratePlacePose 生成放置姿态
GeneratePose 生成目标姿态

Propagating Stages（传播阶段）

  • 从输入状态生成新的状态，并向前或向后传播
  • 特点
    • 有输入状态
    • 生成新的运动轨迹
    • 可以修改机器人状态
  • 前向传播： 从起始到目标状态
Stage 作用
MoveTo 移动到目标
MoveRelative 相对移动
ComputeIK 计算逆运动学
ModifyPlanningScene 修改场景
  • 后向传播：从目标状态到起始状态反向规划，常用于放置任务规划

Connection Stages（连接阶段）

  • 连接两个状态集合
  • 特点
    • 有两个输入
    • 尝试找到可运行的轨迹
    • 通常使用运动规划器
Stage 作用
Connect 使用规划器连接两个状态

示例 State A ---- Connect ---- State B 如果规划成功： State A → trajectory → State B

Container（容器）

  • 用于组织多个stage
  • 主要作用
    • 组织多个Stage
    • 控制执行顺序
    • 管理规划解（solutions）
    • 构建复杂任务结构

SerialContainer（串行容器）

  • 顺序执行多个stage,前一个stage的结果传给下一个stage

ParallerContainer（并行容器）

  • 同时尝试多个规划路径，保留陈宫的方案

Alternative（方案选择容器）

  • 生成多个方案，从中选择最优

规划器

规划器 规划空间 轨迹特点 是否避障 适用场景
PipelinePlanner 关节空间 自由路径 ✔ 全局移动
JointInterpolationPlanner 关节空间 关节线性插值 ✘ 简单姿态变化
CartesianPath 笛卡尔空间 末端直线 部分 接近/抬起

PipelinePlanner

  • 在关节空间搜索路径
  • 轨迹可能是弯曲的，末端路径不一定是直线
  • 优点
    • 可以避障
    • 可以走复杂路径
    • 可以绕开障碍物
  • 缺点
    • 计算慢
    • 末端轨迹不可控
  • 典型用途：规划机器人到目标区域

JoinInterpolationPlanner

  • 关节线性插值
  • 优点
    • 非常快
    • 稳定
    • 不需要搜索
  • 缺点
    • 不会避障
    • 可能会发生碰撞
  • 用于已知安全的姿态，比如打开或关闭夹爪

CatesianPath

  • 规划末端直线运动
  • 规划过程
    • 沿直线采样点
    • 每个点求IK
    • 检查碰撞
  • 优点
    • 末端直线
    • 轨迹可控
  • 缺点
    • 如果中间IK失败就失败
    • 不会绕障碍
  • 抓取动作
