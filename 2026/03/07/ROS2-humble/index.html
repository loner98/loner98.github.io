tf2

  • 管理和查询坐标变换的核心库，解决机器人系统里不同坐标系之间的位置和姿态转换问题
  • 机器系统中有很多不同的坐标系
    • map：全局地图坐标系
    • odom：里程计坐标系
    • base_link：机器人底盘
    • laser：激光雷达
    • camera_link：相机

核心概念

  • Frame（坐标系）
  • Transform（变换）
    • 两个frame之间的关系包含：Translation（位置（x,y,z）），Rotation（旋转（四元数））
  • 时间（Time）
    • tf2是时间感知的变换系统
    • 机器人在移动，坐标系变化必须带时间戳

主要功能

  • 发布坐标变换
    • 通常通过tf（动态变换）、tf_static（静态变换）发布
  • 查询坐标变换：程序可以从查询两个坐标系之间的变换路径，如camera → maptf2会自动找到路径camera → base_link → odom → map，然后计算最终变换
  • 坐标变换

常用tf2库

库 功能
tf2 核心库
tf2_ros ROS接口
tf2_geometry_msgs geometry_msgs转换
tf2_tools 调试工具

常用工具

  • 查看tf树ros2 run tf2_tools view_frames
  • 查看实时变换ros2 run tf2_ros tf2_echo frame1 frame2
  • 发布静态变换ros2 run tf2_ros static_transform_publisher

静态变换

  • 以下命令向 tf2 发布静态坐标变换，导致 world 坐标系和 mystaticturtle 坐标系之间在 z 轴方向偏移 1 米，且不发生旋转。
  • 在 ROS 2 中，roll/pitch/yaw 分别指绕 x/y/z 轴的弧度旋转。 ros2 run tf2_ros static_transform_publisher --x 0 --y 0 --z 1 --yaw 0 --pitch 0 --roll 0 --frame-id world --child-frame-id mystaticturtle
  • 以下命令将相同的静态坐标变换发布到 tf2，但使用四元数表示旋转 ros2 run tf2_ros static_transform_publisher --x 0 --y 0 --z 1 --qx 0 --qy 0 --qz 0 --qw 1 --frame-id world --child-frame-id mystaticturtle
  • 在launch文件中设置静态转换

shared_ptr和unique_ptr

在 ROS2 里：shared_ptr 管“共享资源”，unique_ptr 管“数据流动”

shared_ptr

  • 特点
    • 多个指针可以指向同一个对象
    • 内部有引用计数
    • 最后一个释放时才销毁对象
  • 常见用法
    • Node创建：auto node = std::make_shared<rclcpp::Node>("my_node");
      • exector、callback等多个地方都要使用这个node
    • 消息回调：

unique_ptr

  • 特点
    • 只能有一个拥有者
    • 不能拷贝、只能移动
    • 更轻量、更高效
  • 常见用法
    • 发布消息（高性能场景）
      • 避免拷贝，提高实时性能
  • ros2支持intra-process communication（进程内通信），使用unique_ptr可以实现消息不复制，直接转移所有权

intra-process communication

  • 同一个进程内的节点通信，不拷贝，直接传指针
  • 默认ROS2通信流程（数据被多次复制，延迟高，CPU占用大）
  • intra-preocess
    • 如果两个节点在同一个进程中，直接传指针

--symlink-install

  • 用符号链接代替拷贝文件
  • 默认情况下，build完文件被复制到install/目录，修改源码需要重新build
  • 添加--symlink-install后，install中的文件只是指向源码的快捷方式，该代码可立即生效
  • C++代码仍需要重新编译，仅限于python
