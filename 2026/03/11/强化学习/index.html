基本概念

  • state
  • action
  • policy
  • state transiction
  • reward
  • trajectory：轨迹是一条状态动作奖励链
  • return
    • discounted return
      • discount rate： γ ∈ (0, 1)
      • 使return变为有限值
      • 平衡远期和近期的收益（0近期收益，1远期收益）
  • MDP(Markov decision process)：马尔可夫决策过程

State Value and Bellman Equation

State Value

    • 含义：：在策略 π 下状态 s 的价值
    • ：第 t 步获得的奖励
    • γ：折扣因子（0~1）
    • E：期望（平均）
    • ：起始状态是
  • 从 s 开始，未来所有奖励的折扣和的期望值

Bellman Equationn

  • 描述了不同状态的state-value函数之间的关系
  • 由两部分组成：即时奖励和未来奖励
  • 每个state都有一个这样的公式

Action Value

  • 表示在某个状态下采取某个动作，然后继续按照策略 𝜋行动时能够获得的期望回报。

Optimal Policy and Bellman Optimality Equation

Optimal Policy（最优策略）

  • 对于一个策略来说，如果过对于所有状态s,该策略的回报期望（state value）均大于或等于其他任何策略，则称该策略为最优策略

BOE

  • 尝试所有动作a,选择期望回报最大的动作

  • 最优状态价值唯一，最优策略和最优轨迹不一定唯一

蒙特卡洛方法

MC basic

  • 核心思想： 通过episode的采样来估计q（s,a），再不断改进策略
    • 不需要环境模型
    • 必须等到episode结束才更新
    • 使用平均回报估计价值
  • MC Basic = 用完整轨迹的平均回报估计 q(s,a)，并用贪心策略不断改进

MC Exploring Starts

  • 核心思想
    • 在每一条 episode（回合）开始时，随机选择一个状态-动作对 (S₀, A₀) 作为起点，然后按照当前策略运行到结束，用回报来更新 Q 值。
    • 强制从各种 (状态 + 动作) 开始 → 全面探索
  • 优点
    • 理论上可以收敛到最优策略
    • 不需要环境模型（model-free）
    • 简单直观
  • ε-greedy
    • 在每一步随机探索
类型 特点
Deterministic（确定性策略） 每个状态只选一个动作
Greedy（贪心策略） 总选 Q 最大的动作
Soft Policy（软策略） 每个动作都有概率被选
  • 典型 Soft Policy：ε-greedy
    • 以 1−ε 的概率选最优动作
    • 以 ε 的概率随机探索
  • Soft policy = 每个动作都有机会被选，从而保证持续探索的策略

随机近似与随机梯度下降

Robbins-Monro 算法

  • 在无法直接计算函数期望的情况下，求方程的根
  • 用“带噪声的观测”逐步逼近方程根的迭代方法，是强化学习更新规则的理论基础
  • 核心思想：
    • x_t：当前估计
    • 𝛼_t：学习率（step size）
    • F(x_t,ξ_t)：带噪声的观测
    • 本质：用“带噪声的样本”逐步逼近真实期望的根
  • 收敛条件

SGD（随机梯度下降）

时序差分方法

  • TD target =“当前奖励+折扣后的未来估计“，用来逼近真实回报的一个可在线更新的目标值
  • TD error = TD target − 当前估计值
    • 预测 vs 现实（带未来估计）之间的差值

n-step Sarsa

  • 要实现n-step Sarsa我们需要经验样本，在时间t时还未被收集，需要等待n个时间步

Q-learning

  • Sarsa只能估计给定策略的action value，必须与策略改进步骤结合以寻找最优策略，Q-learning可以直接估计最优动作值并找到最优策略
  • on-policy和off-policy：用来学习的策略（target policy）和用来采样的策略（behavior policy）是否相同
    • on-policy：用当前执行的策略来学习它本身
      • Sarsa：
      • 这里的按当前策略实际采样出来的动作
      • 如果算法是on-policy，则可以online，但不能使用其他策略生成的预先收集数据。
    • off-policy：用一个策略产生数据，但学习另一个策略（通常是最优策略）
      • Q-learning：
      • 并没有使用实际执行的
      • 而是假设下一步动作总是选最优动作
      • 如果算法是off-policy的，则可以通过online或offline方式实现。
