类与结构体区别

一个类的成员默认为private,struct默认为public。 保留struct是为了兼容C，除了可见性，在技术上两者无任何差别。

static

类外的static表示只给自己用，类内的static表示这个属性与实例无关，是所有对象共有的 在类和结构体之外使用static时，当声明静态函数或静态变量时只会在它被声明的C++文件中被看到 当不需要全局变量时，应尽量使用static，除非需要他们跨翻译单元链接 静态方法不能访问非静态变量

可见性

  • private:自己或友元使用
  • protected:自己和子类
  • public

数组

  • 可在堆或者栈上创建，使用new在堆上创建，在堆上创建的数组访问时因为多了一次内存间接寻址，会影响性能
  • new创建的数组没有方法确定其大小
  • 可以使用std::arry数组，更加安全

字符串字面量

  • 字符串字面量是用引号括起来的字符串序列，在编译时确定，存储在程序的只读数据段中
  • 常见类型
类型 语法 示例 说明
窄字符串 “text” “Hello” char 数组，ASCII/UTF-8
宽字符串 L“text“ L“你好“ wchar_t 数组，宽字符
UTF-8 u8“text“ u8“你好“ char 数组，UTF-8 编码（C++11）
UTF-16 u“text“ “你好” char16_t 数组（C++11）
UTF-32 U“text“ U“你好“ char32_t 数组（C++11）
  • 原始字符串字面量（C++11）：使用R"delimiter(...)delimiter"语法，转义字符不会被处理
  • 连接与拼接
  • 编码前缀与后缀
  • 常见错误

const

  • const与指针：const 在左边修饰数据，在右边修饰指针本身。
  • const引用

mutable

与const一起使用 用在lambda表达式中

构造函数初始化列表

确保成员初始化列表要与成员变量的声明顺序一致 成员变量会默认初始化，在构造函数初始化列表中对其进行赋值，可能会造成性能损失，使用构造函数初始化列表可以避免这种性能损失，使用初始化列表替代默认初始化

创建并初始化对象

初始化基类

  • 派生类对象创建时，必须先构建基类部分，再构建派生类部分
  • 多个基类初始化时，执行顺序按继承顺序，不按初始化列表顺序

new关键字

调用new时将消耗时间，找到一个满足我们需求的足够大的内存，并返回一个指向该内存的指针 new不仅分配内存，还会调用构造函数，new本质上是一个操作符

运算符及其重载

运算符可以看作一个函数 重载不改变运算符的优先级

运算符重载的两种形式

智能指针

自动释放内存 unique_ptr：不支持拷贝，只支持移动 shared_ptr: 是一个共享指针，支持拷贝，也支持移动，底层使用引用计数的方式实现，通过跟踪对象的引用次数来决定何时释放其占用的内存。当引用计数降为0时，对象会被自动销毁。

lambda表达式

  • 基本语法
  • 常见简写
部分 作用
[capture] 捕获外部变量
(parameters) 参数列表
-> return_type 返回类型（可省略）
{} 函数体
  • 捕获方式
写法 含义
[] 不捕获任何外部变量
[x, &y] x 值捕获，y 引用捕获
[=] 所有变量值捕获（拷贝）
[&] 所有变量引用捕获

[*this] C++17 捕获整个对象（拷贝）

[&, x] 默认引用捕获，x 值捕获
